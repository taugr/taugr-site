# Tom Auger (taugr)

Personal website for Tom Auger, built with Astro and published as a static site.

## Commands

```sh
pnpm install
pnpm dev
pnpm build
pnpm preview
pnpm lint
pnpm format
pnpm format:fix
```

## Site Structure

- `/` introduces Tom and highlights featured projects plus recent engagements.
- `/essays/` lists selected essays and tutorials.
- `/projects/` lists products, tools, and research projects.
- `/teaching/` collects learning labs, workshops, and masterclasses.
- `/speaking/` collects recent talks and panels.
- `/about/` gives a short bio and career timeline.
- `/archive/` preserves older writing and notes from the previous blog.

## Content

- Essays live in `src/content/posts`.
- Archived posts live in `src/content/archive/posts`.
- Projects live in `src/content/projects`.
- Teaching and speaking entries are currently defined in their page files:
  - `src/pages/teaching/index.astro`
  - `src/pages/speaking/index.astro`

## Redirects

The site is intended to work on GitHub Pages, so old URL redirects are implemented as static HTML rather than relying on server rules.

- Known old `/posts/...` URLs are generated by `src/pages/posts/[...slug].astro`.
- Old top-level pages such as `/archives/`, `/tags/`, `/aitalk/`, and `/tumo2023/` have static redirect pages.
- `src/pages/404.astro` includes a small client-side fallback for older catch-all paths such as `/tags/...` and `/categories/...`.
- `public/fringeevents.html` preserves the old exact `.html` URL.

`vercel.json` is only useful if the site is also deployed on Vercel.

## Deployment

GitHub Pages deployment is configured in `.github/workflows/deploy.yml`.

The workflow installs dependencies, runs `pnpm run build`, uploads `dist/`, and deploys it with `actions/deploy-pages`.

The site is configured for the custom domain `tau.gr`:

- `astro.config.mjs` sets `site` to `https://tau.gr` for canonical URLs, RSS, and sitemap output.
- `public/CNAME` tells GitHub Pages to serve the built site at `tau.gr`.
- `public/.nojekyll` prevents GitHub Pages from applying Jekyll processing to the static output.
